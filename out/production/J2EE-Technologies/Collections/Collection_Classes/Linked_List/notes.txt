         Notes on LinkedList Class

         In Java, public class LinkedList<E> extends AbstractSequentialList<E> implements List<E>, Deque<E>, Cloneable, Serializable

         ~ It is a list, Deque, queue, Stack (legacy), doubly linked list

         Key Features:
         1. No Contiguous Memory
            Unlike ArrayList, elements are NOT stored in an array, Each node is:
                                                                               A separate object
                                                                               Allocated somewhere in heap
                                                                               Linked via pointers
         Why It extends AbstractSequentialList
         ~ Because Random access is expensive
         ~ Iteration is cheap

💥 Why LinkedList Is Rarely Used in High-Performance Systems

Because modern CPUs love:

Contiguous memory

Cache locality

Predictable access

LinkedList has:

❌ Pointer chasing
❌ Poor locality
❌ More allocations
❌ More GC work

That’s why in real systems, you rarely see it.

Even for queues, people prefer:

ArrayDeque

Custom ring buffers
